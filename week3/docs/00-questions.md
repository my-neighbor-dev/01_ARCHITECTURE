# 스터디 전 생각해볼 질문들

이 문서는 스터디 전에 미리 생각해보면 좋은 질문들을 모아놓은 것입니다.
답변은 각 챕터의 문서에서 확인할 수 있습니다.

---

## 1. 인가와 권한 체크

### 질문 1
> **"Spring Security를 사용하면 모든 권한 체크가 끝나는 걸까요?"**

**생각해볼 점:**
- Role 기반 인가로 "수정 권한이 있는가?"는 체크할 수 있지만, "이 데이터를 수정할 자격이 있는가?"는 체크할 수 있을까?
- 사용자 A가 사용자 B의 데이터를 수정하려고 시도하면 어떻게 막을 수 있을까?
- 같은 권한을 가진 사용자라도, 자신이 만든 데이터만 수정할 수 있어야 한다면?

**관련 챕터**: [02-problem-scenario.md](./02-problem-scenario.md)

## 2. 데이터 소유권 체크

### 질문 2
> **"데이터 소유권을 체크하려면 어떻게 해야 할까요?"**

**생각해볼 점:**
- 모든 수정/조회 메서드에 "이 데이터가 내가 만든 건가?" 체크를 넣어야 할까?
- 도메인마다 소유권 기준이 다르다면? (예: 그룹 단위 vs 사용자 단위)
- 이런 체크 로직을 매번 반복해서 작성해야 할까?

**관련 챕터**: [02-problem-scenario.md](./02-problem-scenario.md), [03-naive-approach.md](./03-naive-approach.md)

---

## 3. 중복 코드 문제

### 질문 3
> **"모든 메서드에 같은 검증 로직을 넣으면 어떤 문제가 생길까요?"**

**생각해볼 점:**
- 수십 개의 메서드마다 똑같은 `if (resource.createdBy != user.id) throw AccessDeniedException()` 코드를 넣어야 한다면?
- 비즈니스 로직과 보안 로직이 섞이면 코드 가독성이 어떻게 될까?
- 검증 로직을 수정해야 할 때 몇 군데를 수정해야 할까?

**관련 챕터**: [03-naive-approach.md](./03-naive-approach.md)

## 4. AOP를 활용한 해결

### 질문 4
> **"비즈니스 로직을 건드리지 않고 보안 로직을 추가할 수 있을까요?"**

**생각해볼 점:**
- AOP(Aspect-Oriented Programming)가 무엇일까?
- 메서드 실행 전에 자동으로 권한 체크를 할 수 있을까?
- Annotation만 붙이면 자동으로 검증이 수행되도록 할 수 있을까?

**관련 챕터**: [04-aop-solution.md](./04-aop-solution.md)

## 5. 전략 패턴으로 유연하게

### 질문 5
> **"도메인마다 다른 검증 로직을 어떻게 처리할까요?"**

**생각해볼 점:**
- 강의는 사용자 단위로 체크하고, 그룹은 그룹 단위로 체크해야 한다면?
- 각 도메인마다 다른 검증 로직을 어떻게 유연하게 처리할 수 있을까?
- Enum을 활용한 전략 패턴으로 해결할 수 있을까?

**관련 챕터**: [05-implementation.md](./05-implementation.md)

### 질문 6

- 검증할 ID를 파라미터에서 어떻게 추출할까?
